1 프로그램 개요

본 프로그램은 파우더 제품의 벌크 최적 제조량을 예측하기 위한 도구입니다.

완제품 BOM(Bill of Materials) 데이터와 SAP 실적 데이터를 기반으로, 각 벌크의 이론 필요량에 과거 로스율을 반영하여 실제로 제조해야 할 최적 수량을 산출합니다.

주요 기능

  • BOM 구조 분석 — 완제품 → 성형물 → 벌크 구조를 트리 형태로 시각화
  • 벌크별 필요 제조량 계산 — 발주 수량 입력 시 벌크별 최적 제조량 자동 산출
  • 로스율 기반 예측 — SAP 실적 데이터 최근 5건의 가중평균으로 로스율 산출 (이상치 영향 감소)
  • 신뢰도 점수 표시 — 표본수·편차·최신성 기반 0~100점 평가 및 등급 배지 (높음/보통/낮음/매우 낮음)
  • ML(머신러닝) 예측 비교 — 제조량 구간별 학습된 로스율과 비교 가능
  • 폐기 데이터 보정 — 폐기된 벌크량을 차감하여 실제 사용량 기준으로 계산
  • 데이터 자동 저장 — IndexedDB에 SAP·환입/폐기 데이터 저장, 페이지 재접속 시 자동 복원
  • 결과 내보내기 — 엑셀(XLSX) 또는 CSV 파일로 다운로드 가능

실적 데이터가 없으면 로스율 계산이 불가하여 최적 제조량이 산출되지 않습니다.

4 예측 계산 방식

4-1. 성형물 필요 수량 산정 (가용재고 차감)

완제품 발주 수량 기준으로 산출된 성형물 필요 수량에서, 현재 보유 중인 성형물 가용재고를 차감하여 실제 제조가 필요한 성형물 수량(최종 수량)을 산정합니다.

최종 수량(ea) = 필요 수량(ea) − 가용 재고(ea)

예시: 필요 수량 30,000ea − 가용 재고 5,000ea = 최종 수량 25,000ea

동일 홋수 내 2-code 다수 보유 시
같은 홋수(완제품) 아래에 2-code(성형물)가 2개 이상 매핑된 경우, 해당 홋수의 모든 2-code 가용재고를 합산하여 차감합니다.
예: 동일 홋수에 2-code A(재고 3,000ea)와 2-code B(재고 2,000ea)가 있으면 가용재고 5,000ea로 합산 처리.

가용재고가 필요 수량보다 많은 경우 최종 수량은 0으로 처리되어 벌크 제조가 불필요함을 나타냅니다.

4-2. 이론 필요량 산출

위에서 산정된 성형물 최종 수량을 기준으로 벌크 이론 필요량을 계산합니다.

이론 필요량(g) = 최종 수량(ea) × 벌크 투입량(g)

예시: 최종 수량 25,000ea × 벌크 투입량 1.9g = 47,500g

4-3. 로스율 산출

SAP 실적 데이터에서 해당 성형물-벌크 조합의 과거 이력을 조회하여 로스율을 계산합니다.

로스율(%) = (실제 투입량 - 표준소요량) ÷ 표준소요량 × 100
  • 과거 이력을 최신순으로 정렬한 후, 유효한 최근 5건의 로스율을 가중평균하여 채택합니다.
    (최신 이력일수록 큰 가중치를 부여 — 예: 5건일 때 최신=5, 다음=4, ..., 가장 오래된=1)
  • 표준소요량 3,000g 이하의 소량 건은 제외합니다. (소량 제조 시 로스율이 비정상적으로 높아지는 패턴이 있고, 현장에서 3kg 이하 잔량은 환입 없이 폐기하므로 폐기분이 투입량에 포함되어 로스율이 왜곡될 수 있기 때문)
  • 표준 투입량은 과거 이력 중 가장 최신의 유효값을 사용합니다. (SAP BOM이 변경된 경우 최신 변경값을 즉시 반영)
  • 폐기 데이터가 있으면 실제 투입량에서 폐기량을 차감하여 보정합니다.
  • 유효한 이력이 없는 경우 로스율과 최적 제조량은 "-"으로 표시됩니다.
최근 5건 가중평균을 사용하는 이유
단일 최근 이력만 사용하면 해당 1건이 이상값인 경우(특수 조건, 측정 오차 등) 예측이 크게 왜곡될 수 있습니다.
최근 5건의 가중평균을 사용하면 최신 현장 상황을 반영하면서도 이상치 1건의 영향을 줄일 수 있습니다.

4-5. 최적 제조량 산출

이론 필요량에 로스율을 반영하여 최적 제조량을 산출합니다.

최적 제조량(g) = 이론 필요량 × (1 + 로스율 ÷ 100)

예시: 57,000g × (1 + 10.59% ÷ 100) = 57,000 × 1.1059 = 63,036g

폐기 데이터 보정 원리
현장에서 폐기된 벌크는 전산상 투입으로 기록되어 있지만 실제로는 사용되지 않은 양입니다.
따라서 폐기량을 투입량에서 차감하여 실제 사용량 기준으로 로스율을 계산합니다.

5 ML(머신러닝) 예측 비교

결과 화면에서 "ML 예측 비교" 버튼을 클릭하면 머신러닝 기반의 로스율 및 제조량을 함께 확인할 수 있습니다.

ML 예측값은 SAP 예측값 바로 아래에 인라인으로 표시되어, 가로 스크롤 없이 한 화면에서 두 예측값을 동시에 비교할 수 있습니다. (ML 값은 보라색으로 구분 표시)

5-1. ML 예측 방식

과거 전체 SAP 실적 데이터 중 이상치를 제거한 42,408건을 Gradient Boosting 머신러닝 알고리즘으로 학습하여, 벌크코드별 + 제조량 구간별 예측 로스율을 산출합니다.

2026년 4월 기준 학습 결과: 벌크 6,131종, 테스트 MAE(평균 절대 오차) 11.91%

Gradient Boosting이란?
Gradient Boosting은 여러 개의 간단한 예측 모델(결정 트리)을 순차적으로 쌓아가면서, 이전 모델이 틀린 부분을 다음 모델이 보완하는 방식으로 학습하는 고성능 머신러닝 알고리즘입니다.
대용량 데이터를 빠르게 처리하면서도 높은 예측 정확도를 보여주어, 산업 현장의 수요 예측과 품질 분석에 널리 사용되고 있습니다.

5-2. ML이 학습에 사용하는 변수

ML 모델은 단순히 과거 로스율만 보는 것이 아니라, 로스에 영향을 주는 다양한 조건을 종합적으로 학습합니다.

학습 변수 설명
실적수량실제 생산된 성형물 수량 (제조 규모)
관리유형제품의 관리 유형 분류 (관리유형별 로스 패턴 차이 반영)
작업장공장·설비가 결합된 작업장 정보 (예: 화성 자동 파우더 성형기2)
벌크 평균 로스율해당 벌크의 과거 평균 로스율
벌크 로스율 표준편차해당 벌크의 로스율 변동성 (편차가 클수록 예측이 어려움)
벌크 로스율 중간값해당 벌크의 로스율 중간값 (이상치에 덜 민감한 대표값)
평균 단위당 표준소요량성형물 1개당 벌크 투입량 평균 (제품 특성 반영)
이력이 1건만 있는 품목의 ML 예측
SAP 방식은 그 1건의 로스율을 그대로 사용하지만, ML은 해당 벌크의 로스율뿐만 아니라 공장, 설비, 제조량 규모, 관리유형 등 조건을 종합적으로 반영하여 예측합니다.
따라서 이력이 적은 품목일수록 ML이 더 안정적인 예측값을 제공할 수 있습니다.
반대로, 그 1건이 정상적인 이력이었다면 SAP 방식이 현장 상황을 더 정확하게 반영할 수 있습니다.

5-3. 제조량 구간별 로스율 경향

제조량이 적을수록 로스율이 높아지는 패턴을 학습하여, 구간별로 다른 예측값을 제공합니다.

제조량 구간 3,000g 5,000g 10,000g 20,000g 30,000g 50,000g 100,000g
로스율 경향 매우 높음 높음 보통 낮음 낮음 낮음 매우 낮음

이론 필요량에 가장 가까운 구간의 예측 로스율이 자동으로 적용됩니다.
예를 들어, 이론 필요량이 57,000g이면 50,000g 구간의 로스율이 사용됩니다.

6-2. 시각적 강조 표시

  • 저신뢰도 행 색상 강조
    • 신뢰도 "낮음" → 옅은 주황 배경
    • 신뢰도 "매우 낮음" → 옅은 빨강 배경 + 실측 권장
  • ML vs SAP 예측 차이 강조
    • 5%p 이상 차이 → 노랑 강조
    • 10%p 이상 차이 → 빨강 강조 (둘 중 하나가 크게 벗어남)
  • 5건 가중평균 그룹 표시 — 이력 모달에서 로스율 계산에 사용된 최근 5건을 외곽 테두리로 강조
  • 가중평균 계산 툴팁 — 평균 로스율 셀에 마우스 오버 시 5건 각각의 가중치와 계산 과정 표시

6-3. 결과 정렬

결과 테이블 상단의 드롭다운으로 정렬 기준을 선택할 수 있습니다.

  • 기본 순서 (BOM 순)
  • 신뢰도 낮은 순 / 높은 순 (저신뢰도 품목 우선 확인용)
  • 제조량 많은 순 / 적은 순
  • 로스율 높은 순 / 낮은 순

6-4. 이력 조회

결과 테이블의 각 행을 더블클릭하면 해당 벌크의 과거 제조 이력을 모달에서 확인할 수 있습니다. (화면의 75% 크기로 확대 표시)
제조일, 지시수량, 실적수량, 표준소요량, 투입소요량, 폐기 정보, 보정 후 투입량 등이 표시되며, 로스율 계산에 사용된 최근 5건은 외곽 테두리로 강조됩니다.

6-5. 결과 내보내기

  • 엑셀 다운로드 — 결과를 XLSX 파일로 다운로드
  • CSV 다운로드 — 결과를 CSV 파일로 다운로드

엑셀 파일에는 화면의 기본 컬럼(성형물/벌크/투입량/필요수량/이론 필요량/평균 로스율/최적 제조량) 외에 신뢰도 관련 5개 컬럼이 추가됩니다:

  • 신뢰도 등급
  • 신뢰도 점수 (0~100)
  • 유효 표본수 (몇 건의 이력이 가중평균에 사용됐는지)
  • 편차(%) — 로스율 표준편차
  • 최신 이력(일전) — 가장 최근 이력이 며칠 전인지

7 참고사항 및 주의점

데이터 업로드 순서
3가지 파일의 업로드 순서는 자유입니다.
단, "벌크 필요량 계산" 버튼을 누르기 전에 표준 대비 실적 데이터가 업로드되어 있어야 로스율이 반영됩니다.
데이터 자동 저장 (IndexedDB)
업로드된 SAP 및 환입·폐기 데이터는 브라우저의 IndexedDB에 자동 저장됩니다.
페이지를 닫았다 다시 열어도 데이터가 자동 복원되므로 매번 재업로드할 필요가 없습니다.
다른 기간의 데이터로 교체하려면 전체 초기화 또는 해당 파일 재업로드를 사용하세요.
BOM만 초기화 버튼
BOM 데이터만 바꾸고 싶을 때 사용합니다. 표준 대비 실적 / 벌크 폐기 데이터는 그대로 유지되므로, 기존 이력을 유지한 채 새로운 발주 건만 계산할 때 편리합니다.
로스율 "-" 표시
해당 성형물-벌크 조합의 유효한 과거 이력이 없는 경우 로스율과 최적 제조량이 "-"로 표시됩니다.
실적 데이터가 축적되면 자동으로 로스율이 산출됩니다.
신뢰도 "매우 낮음" 품목 취급
신뢰도 30점 미만("매우 낮음")인 품목은 표본이 부족하거나 로스율 편차가 큰 경우입니다.
예측값을 그대로 쓰지 말고 실측으로 교차 확인하거나, ML 예측값과 비교 후 판단하세요.

자주 묻는 질문

Q. 투입량이 변경되면 어떻게 하나요?

A. BOM 데이터가 최신 투입량을 반영하고 있으면 자동으로 적용됩니다.
프로그램은 BOM에 기재된 투입량을 기준으로 이론 필요량을 계산합니다.

Q. 폐기 데이터는 얼마나 자주 업데이트해야 하나요?

A. 매주 월요일 지난주 폐기 데이터를 전달받아 업데이트하는 것을 권장합니다.

Q. ML 예측과 SAP 예측 중 어떤 값을 사용해야 하나요?

A. SAP 방식(기본)을 우선 사용하되, ML 예측은 참고값으로 교차 검증하는 용도로 활용하시기 바랍니다.

Q. 소수점 셋째자리 투입량도 정확히 반영되나요?

A. 네, 투입량은 소수점 셋째자리까지 정밀하게 계산에 반영됩니다.
여러 색상의 벌크를 비율로 분배하는 경우에도 정확한 예측이 가능합니다.

Q. 신뢰도 점수가 낮게 나왔는데 믿어도 되나요?

A. 신뢰도 "낮음"(30~49점) 이하는 참고용입니다. 실측 또는 ML 예측과 교차 확인하여 최종 판단하세요. 이력이 축적되면 자동으로 신뢰도가 상승합니다.

Q. 페이지를 닫으면 업로드한 데이터가 사라지나요?

A. 아닙니다. 업로드된 SAP 및 환입·폐기 데이터는 브라우저 IndexedDB에 자동 저장되어 페이지 재접속 시 자동 복원됩니다. 새 데이터로 교체하려면 해당 파일을 재업로드하거나 전체 초기화 버튼을 사용하세요.

Q. 과거엔 최신 1건만 썼는데 왜 5건 가중평균으로 바뀌었나요?

A. 단일 최근 이력이 이상값(특수 조건·측정 오차 등)이면 예측이 크게 왜곡될 수 있습니다. 최근 5건 가중평균은 최신성을 유지하면서도 이상치 1건의 영향을 줄여 더 안정적인 예측을 제공합니다.
