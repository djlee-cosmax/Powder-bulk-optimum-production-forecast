1 프로그램 소개

완제품 발주 수량을 입력하면, SAP 실적 데이터를 분석하여 벌크별 최적 제조량을 자동으로 산출하는 프로그램입니다.

이 프로그램으로 할 수 있는 일
  • 완제품 발주 수량 → 벌크별 필요 제조량 즉시 계산
  • 과거 로스율을 자동 반영하여 실제 필요한 양 산출
  • 신뢰도 등급으로 예측 정확도 한눈에 확인
  • ML 예측값과 비교하여 교차 검증
  • 결과를 엑셀 / CSV로 다운로드

예측의 기본 원리

BOM 기준으로 산출된 이론 필요량에 SAP 실적상의 최근 5건 평균 로스율을 반영하여 최적 제조량을 계산합니다. 폐기 데이터가 있는 경우 실제 사용량 기준으로 보정됩니다.

최적 제조량(g) = 이론 필요량 × (1 + 로스율 ÷ 100)

2 로그인

프로그램 접속 시 가장 먼저 로그인 화면이 나타납니다. 관리자에게 발급받은 계정으로 로그인하면 됩니다.

1아이디 / 비밀번호 입력
발급받은 계정 정보를 입력하고 로그인 버튼을 누릅니다.
2메인 화면 진입
로그인 후 우측 상단에 사용자 이름이 표시됩니다. 관리자 권한이면 이름 옆에 [관리자] 뱃지가 붙습니다.
세션 자동 만료
로그인 후 8시간이 지나면 자동 로그아웃됩니다. 1분 간격으로 세션을 확인하므로, 화면을 켜둔 채 방치하더라도 8시간 경과 시 자동으로 로그인 화면으로 이동합니다. 다시 로그인 후 BOM 데이터를 재업로드하면 됩니다.
계정 발급 / 비밀번호 분실
신규 계정이 필요하거나 비밀번호를 잊은 경우 관리자에게 문의해 주세요.

3 사용 준비 (BOM 데이터 받기)

매번 작업 시 사용자가 직접 준비해야 하는 데이터는 완제품 BOM 데이터 1개입니다.
표준대비실적 / 벌크폐기 데이터는 운영자가 갱신할 때마다 자동으로 받아오므로 사용자가 별도로 업로드할 필요가 없습니다.

방법 1: SAP_BOM조회.vbs 사용 (권장)

가장 빠른 방법입니다. 한 번에 여러 완제품의 BOM을 동시에 조회하여 엑셀 파일로 자동 저장됩니다.

1SAP 로그인
SAP GUI를 실행하고 ERP 시스템에 로그인합니다. APO 시스템에 동시 로그인되어 있어도 괜찮습니다 — 프로그램이 ERP 연결을 자동으로 식별합니다.
2SAP_BOM조회.vbs 실행
바탕화면의 SAP_BOM조회.vbs를 더블클릭하면 입력 폼이 나타납니다.
3완제품 정보 입력
엑셀에서 완제품 코드 / 완제품명 / 발주 수량 3개 열을 통째로 복사하고, 입력 폼 상단에 붙여넣은 뒤 "붙여넣은 데이터로 채우기" 버튼을 누르면 모든 행이 자동으로 채워집니다.
한 건씩 직접 입력해도 되며, + 행 추가 / x 삭제 / Enter 다음 칸 이동이 지원됩니다.
4조회 실행 → 엑셀 자동 저장
조회가 완료되면 BOM 결과가 엑셀 파일로 자동 저장됩니다. 30개 이상 입력해도 SAP 다중입력 페이지가 자동 스크롤되어 한 번에 처리됩니다.
APO + ERP 동시 로그인 환경 자동 대응
업무 사유로 SAP APO와 ERP에 동시에 로그인되어 있어도 프로그램이 ERP 연결을 자동 식별합니다. DEV / QAS / PRD 시스템에 동시 로그인된 경우에는 운영(PRD)을 우선 선택합니다.

방법 2: SAP에서 직접 추출

VBS를 사용하지 않고 SAP에서 BOM 데이터를 TXT 또는 CSV로 직접 추출해도 됩니다. 다른 부서의 SAP 레이아웃 파일도 호환 가능합니다.

BOM 파일 필수 컬럼
BOM 파일에는 아래 컬럼이 반드시 포함되어 있어야 합니다.
  • Lev — BOM 계층
  • MTyp — 자재유형 (FERT, HAL1, HAL2 등)
  • 자재 / 자재내역 — 자재 코드 및 명
  • 투입수량 또는 소요수량 — 둘 중 하나가 있으면 됨
※ 필수 컬럼이 누락된 경우 업로드 시 에러 메시지가 표시되며 진행이 중단됩니다.

4 사용 방법 (Step by Step)

1BOM 데이터 업로드
화면의 보라색 "완제품 BOM 데이터" 버튼을 클릭하고 준비한 BOM 파일을 선택합니다. 업로드 즉시 완제품 목록과 발주 수량 입력란이 나타납니다.
파란색 "표준 대비 실적 데이터", 초록색 "벌크 폐기 데이터" 버튼은 [자동] 표시가 되어 있습니다. 평소에는 손댈 필요 없으며, 자동으로 최신 서버 데이터가 적용됩니다.
2발주 수량 입력
완제품 목록에 발주 수량을 입력합니다. 숫자 입력 시 자동으로 콤마가 적용되며, Enter 키로 다음 칸 이동이 가능합니다.
SAP_BOM조회.vbs로 발주 수량을 미리 입력했다면, 그 수량이 BOM 파일에 함께 들어 있어 업로드만으로 자동 반영됩니다.
3벌크 필요량 계산
화면의 "벌크 필요량 계산" 버튼을 클릭하면 새 창에 결과가 나타납니다.
4결과 확인 및 다운로드
결과를 검토한 뒤, 필요 시 엑셀 또는 CSV로 다운로드합니다. (자세한 결과 화면 사용법은 다음 장 참고)
성형물 가용재고 자동 차감
이미 보유 중인 성형물 재고는 자동으로 차감되어 실제 제조가 필요한 수량만 계산됩니다. 동일 홋수에 2-code(성형물)가 여러 개 매핑된 경우에는 모두 합산하여 차감합니다. 가용재고가 필요 수량보다 많으면 최종 수량은 0으로 처리됩니다 (벌크 제조 불필요).
2-code 성형물이 없는 BOM 구조
일부 품목은 BOM이 9자 완제품 → 1자 반제품 → 3자 벌크 구조로 2-code 성형물 단계가 없는 경우가 있습니다. 이 경우 자동으로 1자 반제품을 성형물 대용으로 사용하여 벌크 필요량을 계산하므로 사용자가 별도 조치할 필요가 없습니다.
초기화 버튼 두 가지
  • "BOM만 초기화" — BOM 데이터만 비우고 자동 동기화된 실적/폐기 데이터는 유지. 다른 발주 건을 빠르게 다시 계산할 때 편리합니다.
  • "전체 초기화" — 모든 캐시 데이터까지 삭제 후 새로고침. 데이터가 이상하게 보일 때 사용하면 서버에서 다시 받아옵니다.
중요
"벌크 필요량 계산" 버튼을 누르기 전에 표준 대비 실적 데이터가 로드되어 있어야 합니다. 평소에는 자동으로 동기화되므로 신경 쓸 필요 없으나, 처음 접속하거나 캐시를 초기화한 직후라면 잠시 기다린 후 진행해 주세요.

5-2. 색상 강조의 의미

결과 테이블의 색상 강조는 행동 가이드 역할을 합니다. 색상을 보고 어떻게 처리할지 빠르게 판단할 수 있습니다.

색상 의미 권장 행동
옅은 주황색 행신뢰도 "낮음" — 이력 부족 또는 편차 큼실측으로 교차 확인
옅은 빨강색 행신뢰도 "매우 낮음" — 예측만으로 판단 위험실측 필수
노랑 강조 (로스율 셀)SAP와 ML 예측 차이 5%p 이상두 값 비교 후 판단
빨강 강조 (로스율 셀)SAP와 ML 예측 차이 10%p 이상신중히 검토 후 결정
보라색 글자ML 예측값 (SAP 값 아래 인라인 표시)참고용으로 활용

5-3. 신뢰도 등급별 행동 가이드

등급 의미 권장 행동
높음표본 충분 + 편차 작음 + 최신 이력예측값 그대로 활용 가능
보통조건 일부 충족활용 가능하나 큰 발주 시 주의
낮음표본 부족 또는 편차 큼참고용. 실측으로 교차 확인 권장
매우 낮음표본 1~2건 또는 편차 매우 큼실측 필수. 예측값만으로 판단 금지

5-4. 결과 정렬

결과 테이블 상단의 드롭다운으로 정렬 기준을 변경할 수 있습니다.

  • 기본 순서 (BOM 순)
  • 신뢰도 낮은 순 / 높은 순 — 저신뢰도 품목을 먼저 확인할 때
  • 제조량 많은 순 / 적은 순
  • 로스율 높은 순 / 낮은 순

5-5. 과거 이력 조회

결과 테이블의 행을 더블클릭하면 해당 벌크의 과거 제조 이력 모달이 열립니다. 제조일·실적수량·표준소요량·투입소요량·폐기 정보·보정 후 투입량을 확인할 수 있으며, 로스율 계산에 사용된 최근 5건은 외곽 테두리로 강조됩니다.

평균 로스율 셀에 마우스를 올리면 5건 각각의 가중치와 계산 과정이 툴팁으로 표시됩니다.

5-6. ML 예측 비교

결과 화면 상단의 "ML 예측 비교" 버튼을 누르면 ML 기반 로스율과 제조량이 SAP 값 바로 아래에 보라색으로 인라인 표시됩니다. 가로 스크롤 없이 한 화면에서 두 예측값을 동시에 확인할 수 있습니다.

SAP 방식은 최근 5건의 가중평균을 사용하고, ML 방식은 더 많은 과거 데이터와 다양한 조건(작업장, 제조량 규모 등)을 학습한 결과를 반영합니다. SAP 방식을 기본값으로 사용하되, 차이가 클 때 ML 값으로 교차 검증해 보시면 됩니다.

6 결과 내보내기

결과 화면 상단의 다운로드 버튼으로 파일을 받을 수 있습니다.

  • 엑셀 다운로드 — XLSX 파일
  • CSV 다운로드 — CSV 파일

엑셀 파일에는 화면 기본 컬럼 외에 신뢰도 상세 정보가 추가로 포함됩니다.

  • 신뢰도 등급 / 점수
  • 유효 표본 수 (몇 건의 이력이 가중평균에 사용됐는지)
  • 로스율 편차(%)
  • 최신 이력(일전)

※ 회의자료나 보고용으로 활용 시 엑셀 다운로드를 권장합니다 (서식 + 신뢰도 정보 포함).

7 자주 묻는 질문

Q. 로그인 후 페이지가 갑자기 로그인 화면으로 돌아갔어요.

A. 로그인 후 8시간이 지나면 자동 로그아웃됩니다. 다시 로그인하면 됩니다. (작업 중인 BOM은 다시 업로드 필요)

Q. 로스율이 "-"로 표시됩니다.

A. 해당 성형물-벌크 조합의 과거 이력이 없는 경우입니다. 실적이 축적되면 자동으로 로스율이 산출됩니다.

Q. 신뢰도가 "낮음" 또는 "매우 낮음"으로 나왔는데 어떻게 해야 하나요?

A. 표본이 부족하거나 로스율 편차가 큰 품목입니다. 예측값을 그대로 쓰지 말고 실측 또는 ML 예측값과 교차 확인하세요. 이력이 축적되면 자동으로 등급이 상승합니다.

Q. SAP와 ML 예측값이 크게 차이납니다.

A. 5%p 이상 차이는 노랑, 10%p 이상은 빨강으로 강조됩니다. SAP 값(기본)을 우선하되, 차이가 크면 행을 더블클릭해 과거 이력을 확인하고 신중히 판단하세요. 이력이 1건뿐인 경우에는 ML이 더 안정적일 수 있습니다.

Q. 표준대비실적 / 벌크폐기 데이터를 직접 업로드해야 하나요?

A. 평소에는 필요 없습니다. 운영자가 새 데이터를 갱신하면 페이지 새로고침 시 자동으로 다운로드됩니다. 특정 일자 데이터로 점검할 때만 임시로 수동 업로드하면 되며, 다음 새로고침 시 서버 데이터로 복원됩니다.

Q. 페이지를 닫으면 업로드한 데이터가 사라지나요?

A. 표준대비실적·벌크폐기 데이터는 브라우저에 자동 저장되어 다음 접속 시 즉시 복원됩니다. BOM 데이터만 매번 업로드하면 됩니다.

Q. SAP에 APO와 ERP가 동시 로그인되어 있는데 SAP_BOM조회.vbs가 잘 동작하나요?

A. 네, 자동으로 ERP 연결을 식별하여 동작합니다. DEV/QAS/PRD가 동시 로그인된 경우는 운영(PRD)을 우선 선택합니다.

Q. 다른 부서에서 받은 BOM 파일을 사용해도 되나요?

A. 네, "투입수량" 컬럼이 없으면 "소요수량" 컬럼을 자동으로 사용합니다. 단, 필수 컬럼(Lev, MTyp, 자재, 자재내역)은 반드시 포함되어 있어야 합니다.

Q. 소수점 셋째자리 투입량도 정확히 반영되나요?

A. 네, 소수점 셋째자리까지 정밀하게 계산됩니다. 여러 색상의 벌크를 비율로 분배하는 경우에도 정확한 예측이 가능합니다.

Q. 데이터가 이상하게 보일 때는 어떻게 하나요?

A. 화면의 "전체 초기화" 버튼 → 페이지 새로고침을 한 번 시도해 보세요. 캐시가 비워지고 서버에서 새로 받아옵니다. 그래도 해결되지 않으면 관리자에게 문의해 주세요.
