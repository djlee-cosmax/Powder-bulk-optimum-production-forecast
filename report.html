1. 프로젝트 배경

1-1. 문제점

파우더 성형실에서 벌크를 사용하여 성형물을 타정할 때, 최적의 벌크 제조량을 산출하는 체계가 없어 다음과 같은 문제가 발생하고 있었습니다.

  • 벌크 제조량이 과다하면 잔량 발생 → 환입/폐기 비용 증가
  • 벌크 제조량이 부족하면 추가 제조 필요 → 생산 지연
  • SAP에서 성형 실적의 벌크 사용량을 직접 확인할 수 없어 데이터 역추적에 많은 시간 소요
  • 로스율이 품목/설비/수량 규모에 따라 다름에도 경험적 감으로 제조량 결정

1-2. 목표

생산운영팀 담당자가 완제품 코드와 발주 수량만 입력하면, BOM 구조에서 필요한 벌크를 자동으로 추출하고, 과거 실적 기반 로스율을 반영하여 최적 제조량을 자동으로 산출하는 프로그램을 개발합니다.

3. 주요 기능

3-1. SAP BOM 자동 조회

  • VBScript + HTA 기반 입력 폼: 완제품 코드와 발주 수량을 한 화면에서 동시 입력 (최대 5개)
  • 발주 수량 입력 시 실시간 콤마 자동 포맷
  • SAP T-Code ZSDR9030 자동 실행 → 결과 TXT 파일 바탕화면 자동 저장
  • TXT 파일에 발주 수량 정보 자동 포함 → 사이트 업로드 시 자동 반영

3-2. BOM 구조 조회

  • TXT/XLSX 파일 업로드 → BOM 트리 테이블 표시
  • 완제품별 페이지 네비게이션 (화살표 버튼)
  • 구분(완제품/반제품/성형물/벌크), 코드, 제품명, 소요수량, 재고, 가용재고, 품질검사, 출고할당, 최종입출고일, 고객사, 연구원 등 표시
  • 최종입고일/최종출고일 없는 항목에 자동 PT 표시

3-3. 최적 제조량 예측

  • 완제품 발주 수량 입력 → BOM에서 성형물/벌크 자동 추출
  • 벌크별 이론 필요량 계산: 필요 수량 x 투입량(g/개)
  • 과거 실적 데이터 기반 로스율 반영 → 최적 제조량 산출
  • 완제품별 페이지 분리 + 엑셀 다운로드 + 최적 제조량 복사 기능

3-4. 로스율 계산 로직

항목계산 방법
표준 투입용량(g/개)표준소요량 / 실적수량
실제 벌크 사용량투입소요량 - 폐기량 (환입/폐기 데이터 반영)
로스율(실제사용량 - 표준소요량) / 표준소요량 x 100%
최적 제조량이론필요량 x (1 + 로스율/100)
필터링 조건: 표준소요량 3,000g 이하 건은 소량 생산으로 로스율 왜곡이 발생하여 제외
적용 기준: 최신 실적 1건의 로스율을 적용 (날짜 내림차순 정렬 후 유효 건 채택)
환입/폐기 매칭: 생산오더 + 작업반(화성=파우더성형실 / 평택=파우더성형실(평택2)) 기준

3-5. 실적 이력 조회

  • 벌크별 필요 제조량 행 더블클릭 → 해당 벌크의 과거 실적 이력 모달 표시
  • 제조일, 지시수량, 실적수량, 표준소요량, 투입소요량, 환입/폐기 반영 여부, 보정 후 투입량, 손실량, 설비 표시
  • 로스율 계산에 반영된 최신 이력 빨간 테두리로 강조

6. 기대 효과

  • 벌크 로스 절감: 데이터 기반 최적 제조량 산출로 과다/부족 제조 방지
  • 업무 효율화: 기존 3개 파일 역추적 → 1개 파일 + 자동 계산으로 시간 절감
  • 의사결정 고도화: 품목/설비/수량별 로스율 데이터 축적으로 근거 기반 판단 가능
  • 생산 계획 정확도 향상: 완제품 발주 수량 입력만으로 벌크별 필요 제조량 즉시 산출
  • 환입/폐기 데이터 체계화: 수기 기록을 디지털화하여 추적 및 분석 가능
